## 原型链继承

#### instanceof : 代表A的原型链上是否存在B的原型
#### 优点： 可以继承父类原型
#### 缺点：如果继承的父类中有引用值，进行修改时（不是重新赋值）所有继承父类的实例都会受到影响

```js
son1.arr.push(6);
son2.arr = [1,2,3,4,5,6]
```
#### 正在研究第二种缺点
---
## 构造函数继承：

#### 优点：继承了父类的引用值，但是修改时互不影响
```js
son1.arr === son2.arr   //false
```
#### 缺点：不可以实现对父类原型的继承
---
## 组合继承
#### 优点：原型继承和构造函数继承的优点
#### 缺点：继承的太多的无用属性
